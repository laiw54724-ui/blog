# Roadmap V3

## 現在

### 發布入口已經成形

- Discord 目前已有：
  - `/動態`
  - `/文章`
  - `/補圖`
  - `/個人資料`
  - `/管理`
  - `/help`
- 前台也已經把貼文、文章、tag 分組基礎整理出來。

### 內容閱讀面已經能用

- 首頁、關於、貼文、文章、搜尋、tag 頁都有基本可用結構。
- 前端正在從「分類導向」轉往「內容與 tag 導向」。

### tag v2 已經有雛形

- `genre / tone / setting / relationship / topic` 已經成為正式 shared spec。
- 前台已經能分開顯示 structured tags 與 free tags。
- Discord 建立流程也已經吃同一套 tag 正規化規則。

## 下一步

### 強化 `/管理`

把它從「可操作列表」升級成真正的日常管理入口。

建議補：

- 只看貼文 / 只看文章
- 更清楚的狀態篩選
- 批次操作後的回饋
- 顯示 slug、公開狀態、建立時間

### 補齊 Discord 建立流程欄位

現在能建立內容，但還偏最小可用。

建議在 modal 補：

- 可見性
- tag
- 是否立即公開
- 必要時加簡短摘要或備註

### 定義 tag 正式規格

把 tag 分成兩層：

- 結構 tag：`genre / tone / setting / relationship / topic`
- 自由 tag：其他主題詞

目前已完成：

- shared tag spec 模組
- structured / free tag 前台分流
- Discord 建立內容時的正規化

### 做 tag 探索頁

讓 tag 不只是側欄和搜尋頁的 cloud，而是可以真的逛。

建議補：

- tag index
- 每組熱門 tag
- tag 頁內的同組推薦 tag

### 大分類單獨頁

把 `日記 / 作品 / 評論書摘 / 遊玩` 做成單獨頁，降級為二級導覽，不回到主導航核心。

建議方式：

- 每個大分類有自己的單獨頁
- 頁面上方提供常用 tag chips
- 點 tag 後做交集過濾

## 之後

### 徹底拆開貼文與文章的 metadata

讓兩者在心智和 UI 上不再只是同資料表不同 type，而是兩種不同創作流程。

### 做內容升格流程

把貼文升格成文章變成正式功能，而不是靠手動重寫。

### 建立更完整的內容後台

如果使用頻率變高，之後可以考慮：

- web admin
- 或更完整的 Discord 管理面板
